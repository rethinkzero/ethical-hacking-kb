# What is Network Scanning?

Nmap is the go-to tool for figuring out what's actually on a network. When I run an Nmap scan, I'm essentially knocking on digital doors to see which ones answer—and more importantly, what they tell me when they do.

Think of a network as a massive apartment building. You know the street address (the network range), but you don't know which apartments are occupied (active hosts), which have their doors open (open ports), or what's happening inside each one (running services). Nmap is the tool that lets me methodically check every apartment, see which doors are unlocked, and peek through the peephole to figure out who's home.

# Understanding Ports

A port is a virtual doorway on a computer that a specific service listens on for incoming connections. When you browse a website, your computer connects to port 80 (HTTP) or 443 (HTTPS) on the web server. When you check email, you might connect to port 143 (IMAP) or 995 (POP3S).

There are 65,535 possible TCP ports and another 65,535 UDP ports on every network-connected device. Most of them are closed—meaning no service is listening. Finding the open ones is like finding the unlocked windows in that apartment building.

Common Ports I Look For During Scans

Port Protocol Service Why It Matters to a Security Tester
21 TCP FTP File transfer—often misconfigured with anonymous access
22 TCP SSH Secure remote access—a potential entry point with weak credentials
23 TCP Telnet Legacy remote access—should not be open; plain-text everything
80 TCP HTTP Web server—potential for web application vulnerabilities
443 TCP HTTPS Encrypted web—still vulnerable to configuration issues
3306 TCP MySQL Database—should never be exposed to the internet
3389 TCP RDP Remote Desktop—common attack vector for ransomware
